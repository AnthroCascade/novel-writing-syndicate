# brainstorm

Explores plot possibilities for undrafted or stuck scenes. Reads existing artefacts, identifies narrative gaps, and proposes options with consequences. Use this when a scene is stuck — the author knows it needs to exist but doesn't know what it does, or knows what it should do but can't find the way in.

## Fortes

No fortes. This is orchestrator-level ideation. The orchestrator reads broadly and thinks about possibilities rather than evaluating existing prose.

## Coordination pattern

Single action. Read context, identify gaps, propose options.

## Context requirements

- Scene synopsis and notes (from Scrivener via scrivener-nav)
- Adjacent scene content (what comes before and after)
- Chapter and part context (what the larger arc needs)
- `artefacts/ledger.md` (if exists — for narrative promises that need paying off)
- `artefacts/arc-tracker/` (if exists — for thread status and dormancy)
- `artefacts/characters/` relevant files (for character state and arc position)
- `artefacts/continuity/` relevant files (for established facts)

## Procedure

1. **Resolve scope.** The author specifies a scene or chapter. Parse the Scrivener binder.

2. **Read everything relevant.** Scene synopsis and notes. Adjacent scenes (full content). Chapter context. KB artefacts. The brainstorm needs to understand what surrounds the stuck point.

3. **Identify the gap.** What does the narrative need here? What promises are outstanding? What character arcs need advancing? What threads are dormant? What does the scene before this one leave unresolved?

4. **Propose options.** For each option:
   - What happens in the scene
   - What it changes (character knowledge, relationships, stakes, situation)
   - What it enables (what the next scene can now do)
   - What it costs (what threads it closes, what opportunities it forecloses)
   - How it interacts with existing artefacts (does it pay off a setup? does it create a new promise?)

5. **Present options as options.** Not recommendations. The author feels which one is right. If none are right, they fertilise the compost.

## Output

Options presented to the author. No files written — brainstorming is conversation, not artefact.

## Settlement condition

Options presented with consequences. The author has material to think with.

## Boundary

Ideation, not decision-making. Does not draft prose, does not choose a direction, does not modify the binder or artefacts. Does not evaluate existing scenes — for that, use `/evaluate`. The brainstorm proposes; the author composts and decides.
